This is the demonstration, test-it-out, kick the tires, see how it works instance of DSpace. DSpace is composed of a core-API, and offers several distinct Web Interfaces.

Program Interfaces

Programmatic Interfaces allow other systems to communicate with DSpace. Some interfaces are read-only to allow for harvesting, others enable content to be submitted to DSpace.

  • REST API Official REST API for DSpace (Jersey based), first released in 4.x.
    • DSpace Documentation on REST API
    • REST Collection Reports
    • REST Metadata Query Reports
  • OAI (Open Archives Initiative) Interface that allows for Metadata and Bitstream (content-file) harvesting, supporting OAI-PMH (Protocol for Metadata Harvest) and OAI-ORE (Object Reuse and Exchange) protocols
    • Supported Metadata Harvesting Formats
    • DSpace Documentation on OAI 2.0
    • More info on OAI-PMH protocol and its usage.
  • SWORD (v.1) (Simple Web service Offering Repository Deposit) Service Document (requires login)
    • More info on SWORD protocol and its usage.
  • SWORD (v.2) Service Document (requires login)
    • More info on SWORD protocol and its usage.
  • RDF Linked Open Data
    • More info on the DSpace RDF Linked Open Data interface and its usage.

What is DSpace

DSpace is an open source repository software package typically used for creating open access repositories for scholarly and/or published digital content. While DSpace shares some feature overlap with content management systems and document management systems, the DSpace repository software serves a specific need as a digital archives system, focused on the long-term storage, access and preservation of digital content.
